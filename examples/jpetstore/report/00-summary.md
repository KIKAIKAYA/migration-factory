# 調査報告書（意思決定者向け）— JPetStore システム移行前調査

> 調査日: 2026-06-12  
> 対象システム: JPetStore（ペットショップ Web アプリケーション）  
> 本書は技術用語を使わず、意思決定に必要な情報に絞って記述する。

## システムの概要

対象は、ペット商品の販売を行う Web システムです。利用者は画面上で商品を閲覧し、カートに入れ、注文を完了できます。アカウント登録・ログイン・注文履歴の確認も可能です。

システムの主な構成要素は以下のとおりです（調査済み資産から確認）。

- **画面**: 16 本の動的画面と 2 本の静的ページ（ログイン・商品閲覧・カート・注文・アカウント管理）
- **業務処理**: 商品カタログ参照・ショッピングカート・注文登録・アカウント管理の 4 業務領域
- **データ保存の仕組み**: 13 種類のテーブル（商品・在庫・注文・利用者情報など）を持つデータベース
- **自動化されたテスト**: 画面遷移の自動確認テストを含む 18 本のテストプログラムが存在する
- **現在のデータ保存**: **起動するたびにデータがリセットされる組み込みデータベースを使用中。本番環境での永続的なデータ保存には対応していない**（10-platform.csv より）

## 移行の課題

このシステムを現代の環境に移行する際には、以下の構造的な問題があります。

### 1. 画面処理の仕組みが約 10 年前に開発停止した製品に依存している

画面の入力処理・画面遷移を担う仕組みの核心部分が、2015 年頃に開発が止まった製品に全面依存しています。この製品はその後のシステム標準（Java の標準仕様の世代交代）に対応しておらず、後継製品も存在しません。

**移行への影響**: この部分を別の製品に置き換えるまで、他のすべての近代化作業（実行環境の更新・アプリケーションサーバーの更新）を進めることができません。画面処理に関わるすべてのプログラム部品（4 つの処理クラス）と、16 本の画面ファイルに埋め込まれたこの製品固有の記述を書き直す必要があります（07-framework.csv より）。

### 2. Web 接続部品が旧バージョンのまま固定されており、サポート期限が切れている

ある接続部品が 2024 年末にサポートを終了したバージョンに意図的に固定されており、現在もセキュリティ更新を受けられない状態です。この固定はシステムの構造上の理由から止むを得ない措置であり、上記「1」の問題が解決されるまで変更できません（03-libraries.csv より）。

### 3. 組み込みデータベースは本番利用不可

現在のデータ保存の仕組みは、起動ごとにすべてのデータが消えるテスト向けの構成です。本番サービスには別のデータベース製品への切り替えが必要です（10-platform.csv:applicationContext.xml行 より）。

### 4. 注文番号の重複が発生するリスクがある

注文番号を管理する仕組みに、複数の注文が同時に処理された場合に同じ番号が割り当てられる可能性があります。現在の環境では顕在化しにくいですが、本番データベースへの移行・同時アクセスが増える環境では問題が発生しえます（09-db.csv:SequenceMapper_cache行 より）。

## リスク

以下のリスクを優先度順に示します。

| 優先度 | リスク内容 | 根拠（CSV） |
|-------|-----------|------------|
| 最高  | 画面処理の仕組みの全面書き直しなしには、他のすべての技術更新ができない | 12-impact-matrix.csv |
| 高    | サポート切れの部品が残存しており、脆弱性対応を受けられない | 03-libraries.csv |
| 高    | 現在のデータ保存では本番運用が不可能 | 10-platform.csv |
| 高    | 注文番号の重複による注文データ破損リスク（本番移行後に顕在化する可能性あり） | 09-db.csv |
| 中    | 移行先のデータベース製品によっては、データ定義の一部修正が必要（予約語の衝突） | 09-db.csv |
| 中    | 本番で使用するサーバー環境が未確定 | 00-asset-check.csv |
| 中    | 業務の設計書類（機能仕様・画面遷移図等）が存在しないため、移行の正確な要件把握に限界がある | 00-asset-check.csv |

## 確認事項

移行の方針決定および詳細見積もりのために、以下を顧客・関係者に確認してください。全 30 件の詳細は `data/13-confirm-items.csv` にあります。

### 顧客に確認（13 件）

1. **本番で動かすサーバーの種類**（候補が 8 種類あるが未確定）
2. **移行先として選ぶデータベースの種類**（現在は未定）
3. **注文処理以外に夜間バッチ処理など定期実行の処理があるか**（ソースに見当たらないが存在の確認が必要）
4. **業務仕様書・画面設計書・テーブル設計書等の設計資料が存在するか**（ある場合は提供を依頼）
5. **現在使用している特定のアプリケーションサーバーがすでにサポート終了しているかどうか**（2 種類のサーバー製品について確認が必要。サポート終了の場合は追加対応が必要）

### 技術検証で確認（14 件）

1. **注文番号の重複リスクの対処**（採番キャッシュの無効化を本番移行前に実施すること）
2. **移行先データベース確定後の SQL 互換性確認**（一部の列名・テーブル名が新 DB では予約語と衝突）
3. **最新のサーバー規格への対応確認**（画面処理部品の置き換えに伴うテスト）

### 運用担当に確認（3 件）

1. **社内ネットワーク制限の有無**（外部ライブラリ取得先を社内サーバーに変更する必要があるか）
2. **CI/CD の認証情報の引き継ぎ方法**（現在の構成に依存した認証設定を新環境に移す手順の整備）
